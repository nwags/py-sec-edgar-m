# Migration Checklist (Wave 7)

This is a planning-first checklist.

## Governance
- [ ] Define canonical external package repository and ownership
- [ ] Define public API versus internal implementation policy
- [ ] Define required release artifacts and documentation

## RC workflow
- [ ] Define RC naming/tag rules
- [ ] Define RC creation workflow
- [ ] Define RC invalidation rules
- [ ] Define promotion evidence requirements

## Stable release policy
- [ ] Define promotion-to-stable criteria
- [ ] Define rejection/rollback handling
- [ ] Define changelog/release-note expectations

## Compatibility
- [ ] Define semver meaning
- [ ] Define compatibility window policy
- [ ] Define public API expansion policy

## Validation
- [ ] Define cross-repo validation matrix
- [ ] Define user-testing gate
- [ ] Define release signoff criteria

## Cleanup
- [ ] Define local compatibility-layer retirement criteria

## Pause point before implementation
Do not approve Wave 7 implementation until governance, RC workflow, promotion, compatibility, validation, user testing, rollback, and retirement criteria are all explicit.
