# Migration Checklist (Wave 7.1)

This is a planning-first checklist.

## Package-side governance
- [ ] Define external package repo ownership and review model
- [ ] Define package-side public API freeze statement
- [ ] Define package-side required docs/artifacts

## Release execution
- [ ] Define RC creation workflow
- [ ] Define evidence bundle structure
- [ ] Define promotion/rejection process
- [ ] Define rollback/incident process

## Cross-repo participation
- [ ] Define repo roles in one shared release cycle
- [ ] Define required validation/signoff artifacts
- [ ] Define blocker rules

## User testing
- [ ] Define explicit start gate for cross-application user testing

## Cleanup deferral
- [ ] Keep cleanup/removal out of scope
- [ ] Define future entry criteria only
